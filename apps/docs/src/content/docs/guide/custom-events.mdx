By default, only `text_delta` events are processed (appended to the assistant message content). Use `onEvent` to handle additional event types.

## Basic Usage

```tsx
const { messages } = useChat({
  api: "/chat",
  onEvent: (event, helpers) => {
    switch (event.type) {
      case "text_delta":
        helpers.appendContent(event.delta);
        break;
      case "tool_call":
        helpers.appendContent(`\n[Calling ${event.tool_name}...]\n`);
        break;
    }
  },
});
```

:::caution
When you provide `onEvent`, it **replaces** the default `text_delta` handling entirely. You must handle `text_delta` yourself by calling `helpers.appendContent()`.
:::

## Helpers

The `helpers` object provides two methods:

### `appendContent(delta: string)`

Append text to the current assistant message's content. This is the most common operation:

```tsx
helpers.appendContent(event.delta);
```

### `setMessages(setter)`

Full access to the React state setter for advanced mutations. Use this when you need to modify messages in ways that `appendContent` doesn't support:

```tsx
helpers.setMessages((prev) => {
  const last = prev[prev.length - 1];
  // Custom mutation logic
  return [...prev.slice(0, -1), { ...last, content: "replaced" }];
});
```

## Lifecycle Callbacks

In addition to `onEvent`, you can use these callbacks for observing the chat lifecycle:

### `onMessage`

Called when a complete message is added (both user and assistant):

```tsx
useChat({
  api: "/chat",
  onMessage: (message) => {
    console.log(`New ${message.role} message:`, message.content);
  },
});
```

### `onFinish`

Called when the stream ends with the full message array:

```tsx
useChat({
  api: "/chat",
  onFinish: (messages) => {
    console.log("Stream complete. Total messages:", messages.length);
  },
});
```

### `onError`

Called when a fetch or stream error occurs:

```tsx
useChat({
  api: "/chat",
  onError: (error) => {
    console.error("Chat error:", error.message);
  },
});
```
