## `useChat<TPart>(options): UseChatReturn<TPart>`

The hook accepts an optional generic parameter `TPart` for custom content part types. Defaults to `ContentPart` (built-in `TextPart | ToolCallPart`).

```tsx
// Default — uses built-in ContentPart
const chat = useChat({ api: "/chat" });

// Custom — extend with your own part types
const chat = useChat<ContentPart | ThinkingPart>({ api: "/chat" });
```

### Options

| Option | Type | Required | Description |
|---|---|---|---|
| `api` | `string` | Yes | SSE endpoint URL |
| `initialMessages` | `Message<TPart>[]` | No | Initial messages to prepopulate the chat |
| `headers` | `Record<string, string>` | No | Extra headers merged into every fetch request |
| `body` | `Record<string, unknown>` | No | Extra fields merged into POST body alongside `message` |
| `onEvent` | `(event, helpers) => void` | No | Custom SSE event handler (replaces default `text_delta` handling) |
| `onMessage` | `(message) => void` | No | Called when a complete message (user or assistant) is added |
| `onError` | `(error) => void` | No | Called on fetch or stream error |
| `onFinish` | `(messages) => void` | No | Called when the stream ends |

### Return Value

| Field | Type | Description |
|---|---|---|
| `messages` | `Message<TPart>[]` | Array of `{ id, role, parts }` |
| `isLoading` | `boolean` | `true` while streaming |
| `error` | `Error \| null` | Most recent error, or `null` |
| `sendMessage` | `(text: string) => void` | Send a user message and start streaming |
| `stop` | `() => void` | Abort the current stream |
| `setMessages` | `Dispatch<SetStateAction<Message<TPart>[]>>` | Direct state setter for programmatic control |

## Types

All types are exported for use in your own code:

```ts
import type {
  TextPart,
  ToolCallPart,
  ContentPart,
  Message,
  SSEEvent,
  TextDeltaEvent,
  ToolCallEvent,
  EventHelpers,
  UseChatOptions,
  UseChatReturn,
} from "@devscalelabs/react-sse-chat";
```

### `TextPart`

```ts
interface TextPart {
  type: "text";
  text: string;
}
```

### `ToolCallPart`

```ts
interface ToolCallPart {
  type: "tool_call";
  tool_name: string;
  argument: string;
}
```

### `ContentPart`

Built-in content part union. Use as a base when defining custom parts:

```ts
type ContentPart = TextPart | ToolCallPart;
```

### `Message<TPart>`

Generic over part type. Defaults to `ContentPart`:

```ts
interface Message<TPart extends { type: string } = ContentPart> {
  id: string;
  role: "user" | "assistant";
  parts: TPart[];
}
```

### `SSEEvent`

```ts
type SSEEvent = TextDeltaEvent | ToolCallEvent;
```

### `TextDeltaEvent`

```ts
interface TextDeltaEvent {
  type: "text_delta";
  delta: string;
}
```

### `ToolCallEvent`

```ts
interface ToolCallEvent {
  type: "tool_call";
  tool_name: string;
  argument: string;
}
```

### `EventHelpers<TPart>`

Generic over part type. Defaults to `ContentPart`:

```ts
interface EventHelpers<TPart extends { type: string } = ContentPart> {
  appendText: (delta: string) => void;
  appendPart: (part: TPart) => void;
  setMessages: Dispatch<SetStateAction<Message<TPart>[]>>;
}
```

### `UseChatOptions<TPart>`

Generic over part type. Defaults to `ContentPart`:

```ts
interface UseChatOptions<TPart extends { type: string } = ContentPart> {
  api: string;
  initialMessages?: Message<TPart>[];
  headers?: Record<string, string>;
  body?: Record<string, unknown>;
  onEvent?: (event: SSEEvent, helpers: EventHelpers<TPart>) => void;
  onFinish?: (messages: Message<TPart>[]) => void;
  onMessage?: (message: Message<TPart>) => void;
  onError?: (error: Error) => void;
}
```

### `UseChatReturn<TPart>`

Generic over part type. Defaults to `ContentPart`:

```ts
interface UseChatReturn<TPart extends { type: string } = ContentPart> {
  messages: Message<TPart>[];
  isLoading: boolean;
  error: Error | null;
  sendMessage: (text: string) => void;
  stop: () => void;
  setMessages: Dispatch<SetStateAction<Message<TPart>[]>>;
}
```
